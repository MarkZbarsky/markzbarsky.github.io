Line Follower Autonomous Vehicle

Embedded Systems • September 2024

Overview

For my ENGR 2050 coursework, I designed and implemented an autonomous navigation system centered around line-following capabilities. The hardware foundation consisted of an Arduino Uno microcontroller interfaced with dual infrared sensors strategically positioned to detect optical contrast between a navigation line and the surrounding surface. I developed the control algorithms to process sensor inputs and translate them into precise motor commands for a differential drive system powered by independent DC motors.

System Architecture

The autonomous vehicle employs a differential drive configuration where two DC motors independently control the rear wheels. This design provides the maneuverability necessary for sharp directional changes while maintaining stability during straight-line motion. The infrared sensors operate on the principle of reflectance measurement, detecting the contrast between dark navigation lines and lighter background surfaces.

Control Algorithm Development

The most technically challenging aspect involved tuning the control parameters to handle dynamic path conditions, including sharp directional changes that approached the vehicle's minimum turning radius. I implemented a proportional control system that adjusts motor speeds based on sensor readings, enabling smooth trajectory corrections without oscillation.

The algorithm processes sensor data in real-time, making steering decisions based on the following logic:

  • Both sensors detect the line: Move forward at full speed
  • Left sensor only detects line: Reduce left motor speed, turn right
  • Right sensor only detects line: Reduce right motor speed, turn left
  • Neither sensor detects line: Execute search pattern to relocate the path

Testing and Optimization

Through iterative testing and refinement, I optimized the response characteristics to achieve reliable navigation across complex course layouts. The testing process involved gradually increasing track complexity, starting with gentle curves and progressing to sharp turns and figure-eight patterns.

Sensor Calibration

Initial testing revealed inconsistent readings due to ambient lighting variations and surface irregularities.

Solution: I implemented adaptive thresholding that automatically adjusts sensitivity based on baseline readings taken at startup.

Sharp Turn Navigation

The vehicle initially struggled with abrupt directional changes, often overshooting the line or stopping completely.

Solution: I fine-tuned the proportional control gains and added a brief reverse motor command when both sensors lose the line simultaneously.

System Limitations

Despite successful overall performance, the vehicle encountered limitations with extremely sharp angular turns, such as narrow triangular paths with acute angles. The physical turning radius of the differential drive system represents a fundamental constraint that would require more complex maneuvering strategies or alternative steering mechanisms to overcome. These limitations highlighted the importance of understanding mechanical constraints when designing autonomous navigation systems.

Results and Performance

The completed autonomous vehicle successfully navigated complex paths including sharp corners, S-curves, and closed-loop circuits. The system demonstrated robust performance across different lighting conditions and track configurations. This project provided hands-on experience with embedded control systems, sensor integration, and real-time decision-making algorithms in autonomous vehicle applications.

The final implementation achieved consistent line-following accuracy with minimal oscillation, successfully completing all test courses within the specified time constraints. The experience reinforced the importance of systematic testing and iterative refinement in embedded systems development.
